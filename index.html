3D Parkour Showcase Game

A playable 3D parkour showcase game made in Godot. The game focuses on fast movement, flow and testing different parkour mechanics inside an overgrown city environment. The player can run, jump, slide, wallrun, climb ledges and use rope swinging to move through the level.

Project Overview

This project is a 3D parkour showcase game made to show the movement systems, level design and gameplay ideas I developed during the project. The main focus of the game is movement. I wanted the player to move through the world in a way that feels fast, responsive and fun.

The game takes place in an overgrown city-style environment. The level was first made as a simple greybox testing area, but later I added buildings, materials, assets and atmosphere to make the world feel more alive.

The player moves through a timed route with obstacles, jumps, walls, low spaces and dangerous areas. The level includes a timer and death triggers, so the player can try to complete the route faster and improve their run.

How To Play

The goal is to move through the level as fast as possible using the parkour mechanics. The timer tracks your run, and if you fall into a death trigger you will be reset so you can try again.

Move

Use WASD to move around the level. Move the mouse to control the camera and look around.

Run & Jump

Use Shift to run and Space to jump. Press Space again while in the air to double jump.

Slide & Crouch

Use Ctrl to crouch or slide. Sliding gives the player a small speed boost and gives more momentum when going downhill.

Wallrun

Jump towards a wall while moving forward to start wallrunning. This helps the player cross gaps and keep movement flow.

Ledge Climb

When the player reaches a ledge, the character can climb up if there is enough space above it.

Rope Swing

Hold Space near a rope to grab it and swing across gaps.

Main Features

Basic Movement

The player can walk, run and jump. These mechanics form the base of the player controller and are used throughout the whole level.

Advanced Movement

The game includes crouching, sliding, double jumping, wallrunning and ledge climbing. These mechanics make the movement more varied and give the player more ways to move through the level.

Sliding Momentum

Sliding gives the player a small speed boost. Sliding downhill also gives extra momentum, which makes slopes more useful and makes the movement feel faster.

Rope Swinging

I experimented with rope swinging mechanics. I made a simpler scripted version and also tested a more physics-based version to see which approach worked better.

Timer & Death Triggers

The level includes a timer to track the player’s run and death triggers that reset the player when they fall out of the level.

Camera & UI

The game uses a third-person camera with a SpringArm to reduce camera clipping. I also added a main menu and pause menu to make the game feel more complete.

Custom Player Model

I created my own player model in Blender, rigged it, animated it and imported it into Godot. The character uses animations connected to movement states.

Sound Design

I added footsteps, ambient sound and wind/speed sounds. These sounds give the player more feedback and make movement feel less empty.

Tools Used

The game was made in Godot. I used Godot for the player controller, movement systems, camera, UI, level logic, timer, death triggers and sound implementation.

I used Blender to create the player model and learn more about 3D modelling, rigging and animation. I also used online tutorials, documentation, Reddit and AI support when I got stuck or needed extra explanation.

Development Process

I started by choosing Godot as my game engine and learning the basics of the editor, GDScript and 3D scenes. After that, I built a simple greybox level where I could test the movement mechanics without focusing on visuals yet.

I first worked on basic movement and camera control, then added more advanced mechanics such as sliding, wallrunning, double jumping, ledge climbing and rope swinging. I tested these mechanics inside the level and adjusted values like speed, gravity, momentum and air control to make the movement feel better.

Later, I created a player model in Blender, rigged it, animated it and imported it into Godot. I also added a main menu, pause menu, sounds, a timer, death triggers and visual assets to turn the level into a more complete playable showcase game.

Visual Direction

The visual direction of the game is inspired by overgrown and post-apocalyptic city environments. I wanted the world to feel abandoned but still interesting to move through. Earlier in the project I thought about a jungle-style game, but I later changed direction to an overgrown city because it fit better with the parkour idea.

The level started as a greybox with simple blocks. Later I added buildings, nature assets, textures and materials to make the world feel more alive. The final look is not meant to be fully realistic, but it gives the level more atmosphere and makes the route feel more like an actual game world.

What I Learned

During this project I learned how to build a 3D game in Godot from the ground up. I learned how scenes and nodes work, how to write GDScript, how to use collisions and raycasts, and how to build a movement system that connects to a camera and animation setup.

I also learned a lot about Blender. I learned how to create simple 3D models, how to use modifiers, how to rig a character and how to create basic animations. Importing the model into Godot taught me that scale, rotation, hierarchy and animation setup are very important.

The biggest thing I learned is that a mechanic can work technically but still not feel good in gameplay. Because of that, testing and adjusting values was a big part of the project.
