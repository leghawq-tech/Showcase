3D Parkour Movement Prototype

A 3D parkour game prototype made in Godot. The goal of this project was to create a playable movement prototype with running, jumping, sliding, wallrunning, ledge climbing and rope swinging.

Project Overview

This project is a playable 3D parkour prototype. The focus was not to create a fully finished game, but to build a working prototype where different movement mechanics could be tested in one level.

The player moves through an overgrown city-style level and can use different parkour mechanics to complete the run. The level includes a timer and death triggers, so the player can try to finish the route faster.

Main Features

Basic Movement

Walking, running and jumping form the base of the player controller.

Advanced Movement

The prototype includes crouching, sliding, double jumping, wallrunning and ledge climbing.

Rope Swinging

I made experimental rope swinging mechanics to test how swinging could work in the game.

Timer & Death Triggers

The level includes a timer and death triggers to make the prototype feel more like a playable run.

Camera & UI

The game uses a third-person camera with a SpringArm and has a main menu and pause menu.

Sound Design

I added footsteps, ambient sound and wind/speed sounds to make the movement feel more alive.

Tools Used

The game was made in Godot. I used Blender to create and animate the player model. I also used online tutorials, documentation, Reddit and AI support when I got stuck or needed extra explanation.

Development Process

I started by choosing a game engine and learning the basics of Godot. After that, I made a simple greybox test level where I could test movement mechanics without spending too much time on visuals.

I then added basic movement, camera control and more advanced mechanics like sliding, wallrunning and ledge climbing. Later, I created my own player model in Blender, rigged it, animated it and imported it into Godot.

Near the end of the project, I improved the level by adding assets, sound effects, a timer, death triggers and a more complete main menu.

What Could Be Improved

The prototype could still be improved by polishing the animations, cleaning up the player controller code and making the rope mechanic feel smoother. The level could also be made more consistent visually, but for this project the main focus was on creating a working movement prototype.
